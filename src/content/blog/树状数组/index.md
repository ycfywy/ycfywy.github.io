假设有一个长度为 $n$ 的数组，我们需要频繁地做两件事：
- 修改：把第 $i$ 个数加上一个值 $v$（单点修改）。
- 查询：算一算前 $i$ 个数的总和是多少（前缀和查询）。

普通数组：修改 $O(1)$，但求前缀和需要遍历，复杂度 $O(n)$。

前缀和数组：查前缀和 $O(1)$，但一旦修改了第 $1$ 个数，后面所有的前缀和都要跟着改，修改复杂度 $O(n)$。

树状数组则是两者的完美平衡，它能做到修改 $O(\log n)$，查询 $O(\log n)$。

![alt text](image-1.png)


要证明树状数组（BIT）加减 `lowbit` 操作的**完备性**（即不重、不漏、完全覆盖），我们需要从**二进制的数学本质**切入。

树状数组的核心数学定义是：**对于任意正整数 $x$，节点 `tree[x]` 负责管辖的区间是：**


$$(x - \text{lowbit}(x), x]$$


也就是说，它管辖了从 $x$ 开始逆序数 `lowbit(x)` 个元素。这个定义是证明一切的基础。

---

## 1. 证明：为什么“减 `lowbit`”能不重不漏地覆盖所有区间？

我们要证明：从任意一个数 $X$ 开始，不断执行 $X = X - \text{lowbit}(X)$，直到 $X = 0$，沿途所有节点管辖的区间连起来，**恰好拼成 $(0, X]$，既没有缝隙，也没有重叠。**

### 数学证明：

假设 $X$ 的二进制表示里有若干个 `1`，我们可以把它写成非零位权的总和：


$$X = 2^{p_k} + 2^{p_{k-1}} + \dots + 2^{p_1} \quad (p_k > p_{k-1} > \dots > p_1)$$

根据定义，此时 $X$ 最低位的 `1` 对应的值就是 $\text{lowbit}(X) = 2^{p_1}$。
那么，当前节点 `tree[X]` 管辖的区间就是：


$$(X - 2^{p_1}, X]$$

当我们让 $X_1 = X - \text{lowbit}(X)$ 时，其实就是消去了最低位的 $2^{p_1}$。此时：


$$X_1 = 2^{p_k} + 2^{p_{k-1}} + \dots + 2^{p_2}$$


现在 $X_1$ 的最低位变成了 $2^{p_2}$，所以 `tree[X_1]` 管辖的区间是：


$$(X_1 - 2^{p_2}, X_1] \implies (X - 2^{p_1} - 2^{p_2}, X - 2^{p_1}]$$

仔细观察这两次无缝衔接的区间：

* 第一步覆盖：$(X - 2^{p_1}, X]$
* 第二步覆盖：$(X - 2^{p_1} - 2^{p_2}, X - 2^{p_1}]$

**它们的首尾完全连在了一起！** 以此类推，每一次减去 `lowbit`，都在数学上完美地接上了前一个区间的左端点。由于 $X$ 的二进制中 `1` 的个数是有限的（最多 $\log_2 X$ 个），最后一次减法一定会把唯一的、最高位的 `1` 减掉，使 $X$ 变成 `0`。

最终，所有区间首尾相连，拼出来的总区间就是：


$$(0, X]$$


**因为每一次的左端点都精准等于上一次的右端点，所以绝对不重不漏。**

## 2. 证明：为什么“加 `lowbit`”能不多不少地覆盖所有父亲节点？

更新一个点 $A[x]$ 时，我们需要找出**所有管辖了 $x$ 的节点**。
我们要证明：从 $x$ 开始，不断执行 $x = x + \text{lowbit}(x)$，能**穷举所有包含 $x$ 的区间，且一个不落**。

### 数学证明：

什么时候一个节点 $Y$ 能管辖 $x$？根据定义，必须满足：


$$Y - \text{lowbit}(Y) < x \le Y$$

我们直接从二进制的结构来看：
设 $x$ 的二进制最低位 `1` 在第 $i$ 位（即 $\text{lowbit}(x) = 2^i$）。
当我们将 $x$ 加上 $\text{lowbit}(x)$ 时，会发生**进位**。

1. **如果第 $i+1$ 位原本是 `0**`：加法会让第 $i$ 位变 `0`，第 $i+1$ 位变 `1`。
* 新的数 $x' = x + 2^i$。
* 此时 $x'$ 的最低位变成了 $2^{i+1}$（甚至更高）。
* 那么 $x' - \text{lowbit}(x') \le x' - 2^{i+1} = x - 2^i < x$。
* 显然，满足 $x' - \text{lowbit}(x') < x \le x'$。**说明 $x'$ 确实管辖了 $x$！**


2. **如果第 $i+1$ 位原本是 `1**`：加法会引发**滚雪球式的连续进位**，直到遇到第一个 `0` 为止。
* 这就像是在二进制的树形结构里，直接跨过了当前这一层的“堂哥”节点，跳到了更高层的“亲生父亲”或“爷爷”节点。
* 这些通过进位产生的更大数字 $Y$，其管辖区间会因为 $\text{lowbit}(Y)$ 变得极大，从而像一把大伞一样，完美罩住底下的 $x$。



### 为什么不能跳着加？为什么这个顺序是对的？

因为在树状数组中，任何一个包含 $x$ 的“合法上级” $Y$，**其二进制表示的前缀必然与 $x$ 在高位保持一致，而唯一的区别就是 $Y$ 刚好把 $x$ 某个低位的 `1` 通过进位消掉了。**

执行 $x = x + \text{lowbit}(x)$ 的本质，就是**从右往左，依次把 $x$ 的各个 `1` 向上进位变成 `0` 的动态过程**。

* 它是唯一一条能够从小区间、逐步扩大到大区间的“提拔路径”。
* 如果你跳过某一步（比如加了别的数），高位就会被破坏，区间的左端点就会移动，从而漏掉或者错误地包含其他不相关的区间。

## 💡 终极直观理解

* **减 `lowbit` 的本质是“拆分”**：把一个大数字 $X$，按照二进制位，切成一个个互不重叠的小砖块（长度为 $1, 2, 4, 8\dots$），直到把 $(0, X]$ 铺满。
* **加 `lowbit` 的本质是“冒泡”**：你在水底（位置 $x$）放了一个无形的气泡，它每一次向上游动（进位），气泡管辖的体积（`lowbit` 长度）都会翻倍变大，直到变成覆盖全海域的超级大浪，沿途碰到的所有容器（父亲节点）都会被它精准充满。
