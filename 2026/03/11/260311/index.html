lora微调的原理是什么

当我们对模型进行微调时，参数的变化量可以表示为 ΔW。LoRA 的理论基础源于一个观察：模型在特定任务上的内在秩（Intrinsic Rank）其实是很低的。也就是说，参数的变化量可以分解成2个low rank矩阵，这就大大减少了微调需要动的参数量。

h = W0x + ΔWx = W0x + BAx

矩阵 A 通常采用高斯分布随机初始化。矩阵 B 初始化为全 0。冻住W0，只更新AB

Transformer的核心思想是什么

自注意力机制，并行计算序列中每个token与其他token的关联程度。

对于 (seq, dim)做矩阵运算，得到qkv (seq, v_dim) 然后做一个q@k 得到(seq, seq)的得分矩阵，经过softmax作为权重乘上v

transformer和llama的LN有什么区别

transformer的LN是post-norm，也就是经过attention 以及 feed forward之后来一个 add and norm。且ln操作是减去均值除以标准差

llama是pre-norm，在输入attention之前就做ln。且不减去均值。

解释一下post-norm和pre-norm为什么这样选？

$$\text{LN}(x) = \frac{x - \mu}{\sigma}$$

$$\frac{\partial \text{LN}(x)}{\partial x} \approx \frac{1}{\sigma}$$

显然输入向量 x 的波动越大（σ 越大），LayerNorm 传回来的梯度就越小。当遇到大的参数更新的时候，这种跳动会显著增加下一层输入的方差 σ，LN就会抑制其传播。

那么pre-norm呢？有一条高速的路径传递梯度

xl + 1 = xl + Sublayer(LayerNorm(xl))

$$\frac{\partial \mathcal{L}}{\partial x_l} = \frac{\partial \mathcal{L}}{\partial x_{l+1}} \cdot \mathbf{(1 + \text{Sublayer}'(\text{LayerNorm}(x_l)) \cdot \text{LN}'(x_l))}$$

bert和gpt有什么区别

bert是encoder-only，做的事情是类似于完形填空的。 gpt是decoder-only，做的事情是predict next token。

为什么decoder-only成为主流？

  • 不需要cross attention，计算开销更低
  • predict next token更符合逻辑，且任务更统一，就是根据过去预测现在。

RLHF和 DPO是什么? DPO的loss是什么?

RLHF (Reinforcement Learning from Human Feedback)

  • SFT:在高质量问答数据上训练。
  • 训练奖励模型: 让模型看两个回答（A和B），人类告诉模型哪个更好。模型据此学出一个打分系统。
  • PPO 强化学习：利用奖励模型给出的分数，通过强化学习（PPO 算法）不断调整大模型，使其分数越高越好。

DPO $$\mathcal{L}_{DPO}(\pi_{\theta}; \pi_{ref}) = -E_{(x, w, l) \sim D} \left[ \log \sigma \left( \beta \log \frac{\pi_{\theta}(w|x)}{\pi_{ref}(w|x)} - \beta \log \frac{\pi_{\theta}(l|x)}{\pi_{ref}(l|x)} \right) \right]$$

$$\text{Term A} = \beta \log \frac{\pi_{\theta}(y_w|x)}{\pi_{ref}(y_w|x)}$$这代表当前模型（πθ）相对于原始模型（πref），给人类喜欢的回答 (yw) 分配了多少更高的概率。数值越大，说明模型进步越明显。第二项：坏回答的“退步程度”$$\text{Term B} = \beta \log \frac{\pi_{\theta}(y_l|x)}{\pi_{ref}(y_l|x)}$$这代表当前模型相对于原始模型，给人类讨厌的回答 (yl) 分配了多少概率。我们希望这个值越小越好。最终对比：拔河结果Gap = Term A − Term BDPO 的目标就是让这个 Gap（差距） 越大越好。外层套了一个 Sigmoid (σ) 和  − log ，这在数学上等同于：如果 Gap 变大，Loss 就变小。

数据清洗流程

  • 原始数据抓取：网页抓取、PDF、代码库等等，统一解析并编码成utf-8
  • 启发式过滤：长度、统计学过滤（符号比例 词汇分布 困惑度）
  • 去重：文本相似度、Hash等
  • 质量评估：分类器过滤、有害性过滤等等
  • 采样配比：高质量多遍，低质量少一些。

大模型结构有哪些变化

  1. Encoder-Decoder架构： 以Bert为代表

具体来说，就是按照Transformer原始论文的架构：input过N个Encoder Block形成K V（每一个Encoder Block是什么？就是原始x过了input embeddings之后，来个Attention 然后过一个FeedForward->就是一个2层的mlp 先升维再降维。当然所有的都还有add and norm不要忘记）。output过一个Masked Attention形成Q，然后和K V做Attention。然后经过FeedForward。最后来一个hidden dim 到 vocab_size的linear以及softmax即可。

  1. Decoder-only架构：以GPT系列、Qwen等等现在主流的Dense model

其实就是取消了Encoder：所以我们不需要Encoder的Attention以及Decoder中的第二个Attention。这里输入x之后，直接加上embeddings之后，自己做Attention。但是这里要做Masked Attention（逻辑上，LLM是predict next token 不能让他看到后面的内容。)

  1. Moe架构: 以Deepseek-r1等等

对Dense model来说，每输入一个 Token，模型中所有参数都要参与计算。随着输入length增加，计算成本也会增加。因此我们希望对Transformer Block进行改进，不要全量计算，而是只激活一部分参数。

考虑到改进，其实只能动一动FFN层。因此就引入了Moe架构：也就是加了门控的FFN。

其实就是对每个token，假设dim = 1024，expert_num = 8。也就会过一个1024*8的矩阵，得到长度为8的digits向量。过一个softmax就是每个token对每个专家的routing score. 假设我们选择top-k个专家，那就重新给routing score做一个归一（屏蔽掉别的分数）。

每一个专家，都是一个FFN(先升维度 再降维度) 假设top-k = 2那么你的token就会分别通过两个ffn得到 dim->hidden dim -> output dim的向量。最后再根据routing score加权就可以了。

  1. 主流模型在Transformer上的一些变化
  • Pre-Norm： 原始模型将 LayerNorm 放在残差连接之后（Post-Norm），这在深层网络中容易导致梯度不稳定。现在的模型几乎全部采用 Pre-Norm，即将归一化放在注意力层和 FFN 层之前，这让训练更加稳定。
  • RMSNorm: 传统LayerNorm其实就是让数据的分布变成01正太分布。所以是x-均值/标准差。但是RMS只除以标准差（似乎也很稳定）。减去均值相当于平移不变性，除以标准差相当于缩放不变性。
  • Rope: 不再用Absolute Embedding，且不再直接加到x上。而是加到KV上。
  • SwiGLU / GELU：不再用Relu改用SwiGLU或者GELU。FFNSwiGLU(x) = (SiLU(xW1) ⊗ xW2)W3 $$f(x) = x \cdot \sigma(x) = \frac{x}{1 + e^{-x}}$$f′(x) = σ(x) + x ⋅ σ(x)(1 − σ(x))。SiLU作为激活函数更好，无上界： 当 x → ∞ 时，f(x) ≈ x。这保证了在高激活区域不会出现梯度饱和（Gradient Saturation），从而加快收敛。有下界： 具有类似正则化的效果，防止输出值无限发散。SiLU的输出类似于一种门控。其数值范围是[ ≈  − 0.278,  + ∞)。相比于RELU只能丢弃掉小于0的值，SiLU更灵活。
  • GQA：KV Cache的显存消耗，改成一个多个Q对应Q / k个 KV。
