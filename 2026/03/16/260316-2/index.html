10.手撕：乘积最大子数组（LeetCode 152） ## AI Learning

GRPO 和 PPO 在 RLHF 中的核心区别是什么？

PPO的Advantage是根据TD从后往前算出来的，是token level的，粒度更细。且TD需要一个Value model预测当前state的Reward。GRPO的Advantage是组内平均的，是基于采样的。

KL 散度在 RLHF 中的作用是什么？它的公式如何直观理解？

KL散度：防止模型做Reward hacking。且约束模型不要和ref差的太多（忘记掉之前学习的知识）

vLLM 的核心优势是什么？它是如何通过 PagedAttention 提升显存利用率的？

vLLM原始论文是Page Attention。过去的GPU Memory管理是直接分一个固定大小的Block。现在借鉴OS中的虚拟内存的思路，按需分配，做虚拟页到物理页的映射。举个例子，预先给一个User Request分配2个page，当他用完的时候，再申请新的Page。

有什么好处？

  • 按需分配
  • 方便同一Prompt多推理（可以共用最开始的kv cache）

微调数据集的结构是怎样的？有多少条？数据清洗和有效性验证是怎么做的？

  • alpaca: instuction input output
  • openai: role = “user” “system” content

数据清洗：

  • 长度
  • Embedding语义去重
  • 过一些打分model 去掉分数地的

有效验证：

  • LLM as judge
  • 随机采样 小规模跑一下，看看loss稳定不稳定

奖励函数是如何设计的？

  • 看目标任务：计算题要做对 程序题要能通过test case –> rule-based reward
  • 对只有最后有reward的要细化 或者设计一些和目标不一样 但是对目标有帮助的：射击游戏（要活着 、要探索地图、 不应该不动、 长时间不杀人就给负的reward……） –>Reward Shaping
  • 多个角度：回答的有用性、安全性、逻辑性等等 –>Preference Reward

怎么避免模型靠堆字数刷奖励？

给一个长度的正则化reward，比如r - alpha * length。

KV Cache 在长上下文推理中可能被污染，你们有做缓存隔离或清理机制吗？

缓存隔离 什么鬼？

倒是KV Cache的管理可以讲一讲：

  • sliding window 只关注比较近的KV
  • 清除远的KV 只保留关键token（标点符号 关键词 等等）
  • KV融合压缩 较少KV保留原有信息
