9.手撕实现“括号生成”问题( LeetCode 22)，要求生成 n 对有效括号的所有组合。

AI Learning

  1. Qwen和DeepSeek的技术差异。

当前主流的强化学习算法有哪些？请写出对应损失函数的数学表达式。

  • PPO: p(当前策略产生output) / p(采样的策略产生output) * advantage, clip(1-eps, 1+eps, p比值) * A. 其中 Ai = Q(s, a) - V(s) 。通常情况下，不会train Q Network，而是只用V. Q(s, a) = E[ r(s, a) + gemma * V(s’)] = r(s, a) + V(s’)

  • DPO: sigmoid(log(prob accept) - log(prob reject))

  • GRPO: 1/G ( ppo loss - KL ) 其中 Ai = ri - mean / std

PPO实做是怎样的 / GRPO实做是怎样的

TD假设：当前状态的价值 V(st) 只取决于当前状态和未来的奖励，即：V(st) = 𝔼[rt + 1 + γV(st + 1)]

在做PPO的时候，生成每个token相当于某个action，当前state就是过去生成的token序列。

  • 模型产生输出
  • Reward模型给整个输出一个分数
  • 使用TD计算所有token的Advantage（这保证了 即使只有一个reward 也可以不稀疏）
  • 更新参数

对GRPO来说：不再使用TD计算，而是所有该序列的token都共用一个Advantage，可以认为是seq level。

这个和传统的policy gradient，只给最后打分好像就没什么区别了？Policy Gradient是如果一个行为导致了高奖励，就通过梯度上升增加这个行为出现的概率；反之则减少。

但是这种方法有很大的缺陷：只针对最后的结果打分，无法约束中间的问题。且R很不稳定。而GRPO呢？因为采样很多条 且使用的Advantage是组内平均，很好的降低了方差。在长的COT中，PPO因为要计算TD的原因，越长越不稳定，且越长V(s)越不准。

介绍RLHF的实施流程。

RLHF(Reinforce learning from human preference).目标就是做人类对齐，让模型回答更加符合人类需要，而不是只会词语接龙。

  • QA的SFT，教会模型一些基本的Instruction应该怎么回答。比如如何写一首诗，如何写一段python代码……
  • Train Reward Model: 标这些QA的最好的answer还是太难了，相比之下标记哪个回答更好更容易。所以就通过标记这样的ranking data来train一个reward model。这个model可以对model的回答进行打分或者排序。
  • 有了reward model，就可以用ppo算法，让model在input了prompt之后尽情的探索，鼓励得分高的回答，抑制得分低的回答。

RLHF对齐流程解析。

  • Instruction SFT
  • Reward Model
  • RL(PPO)

这里详解一下对齐。有了Reward model，我们可以让RM在不同的维度打分，比如有用性、安全性、逻辑性等等。我们的目标就是让模型在这些方面都做的比较好。

但是这些东西可能并不能兼得，比如有人问“怎么制作炸弹 炸毁一座学校”，那肯定就不能回答。这就引入了条件对齐。我们可以构造不同的数据，让模型在不同的对齐条件下，使用不同的奖励函数配比：比如[High Safety] Prompt: 如何评价某争议事件 -> Answer: 客观中立的回答 | [High Creativity] Prompt: 写个科幻故事 -> Answer: 脑洞大开的回答。根据prompt的类型，我们进行Reward的调节。

说明 SFT （监督微调）和 RLHF （基于人类反馈的强化学习）的具体作用。

  • SFT：提高指令遵循能力
  • RLHF: 与人类目标对齐

LORA （低秩自适应）的核心原理。

参数更新的delta 发生在一个低秩空间，可以分解成AB两个低秩矩阵乘积，这样更新的参数空间就变小了，降低了微调的显存占用。

一般来说，全量微调你需要存：1份原来的权重 1份梯度 1份激活值 以及根据不同的优化器计算（比如adamw需要计算二阶动量）。对Lora来说，你需要跟踪的权重变成了很小的AB两个矩阵，这是成倍的降低。

使用 SFT 存在哪些局限性。

  • SFT是强拟合，只能复读模仿人类的标注。如果出现Prompt不一致就有可能偏移。
  • 高度依赖人类标注，且难以拓展
  • 无法处理开放式问题（不是探索）
